## Future Sync Workflow Expectations

The future split-repo workflows should remain inactive until the repositories
are created and reviewed. When activated, each workflow should:

- run on pushes to `main`;
- be path-scoped to the owning `volume-*` directory;
- check out `wsollers/Learning-Real-Analysis`;
- mirror only the owning volume directory into the monorepo;
- use `rsync --delete` only after dry-run review;
- commit with `chore: sync volume-* from lra-volume-*`.

Expected future mirrors:

- `lra-volume-vi`: `volume-vi/` to `Learning-Real-Analysis/volume-vi/`
- `lra-volume-vii`: `volume-vii/` to `Learning-Real-Analysis/volume-vii/`
- `lra-volume-viii`: `volume-viii/` to `Learning-Real-Analysis/volume-viii/`

`rsync --delete` is destructive full-replace sync. Before activation, run a
dry run and inspect every delete/create/update path.

Inert template files are prepared at:

- `docs/templates/sync-to-monorepo-volume-vi.yml`
- `docs/templates/sync-to-monorepo-volume-vii.yml`
- `docs/templates/sync-to-monorepo-volume-viii.yml`

These template files are not active workflows.

## Safe Creation Order

1. Create empty private or public repositories under `wsollers` with default
   branch `main`.
2. Populate each repo from the copy manifest above.
3. Add `SYNC_PAT` only after the initial repository content has been reviewed.
4. Add `.github/workflows/sync-to-monorepo.yml` from the matching inert
   template.
5. Run the workflow in dry-run mode first.
6. Review the dry-run `rsync --delete` output.
7. Enable write sync only after the reviewed dry run matches expectations.
8. Build each split repo locally or in CI before treating it as active.

## Dry-Run Validation Checklist

- The target repo exists under the expected `wsollers/lra-volume-*` name.
- The target repo has default branch `main`.
- Only one volume directory is present in each split repo.
- `common/`, the volume-owned bibliography shard, `constitution/`, and `docs/`
  are present.
- Required image asset is present:
  - `images/euler.png` for Volume VI;
  - `images/newton.png` for Volume VII;
  - `images/hilbert.png` for Volume VIII.
- The volume repo `main.tex` inputs only its own volume index.
- Workflow dry-run output touches only the intended monorepo destination.
- No optional non-target images are copied unless explicitly approved.
- No theorem labels, proof labels, citations, or cross-references change.

## Rollback Plan

If repository creation or dry-run sync is wrong:

1. Do not enable write sync.
2. Disable or delete the new repo workflow.
3. Revert any monorepo commit created by a failed sync.
4. Delete or archive the newly created split repo only after confirming no
   unique content was authored there.
5. Recreate from the manifest after the mismatch is understood.
